Caractéristiques

VB-like
Xojo s'appuie sur une variante propriétaire du langage Visual Basic. C'est d'ailleurs une sorte de suite à VB6, avec les mêmes philosophies en terme de création d'interface et de gestion de projet.

Format propriétaire
Xojo se programme avec son IDE. Les formats de fichiers sont propriétaires et le projet n'est pas tout texte comme d'autres langages de programmation ou framework.

EDI multiplateforme
L'EDI de Xojo est disponible sur Windows, Mac et Linux. L'EDI est programmé par Xojo lui-même.

Cible multiplateforme
Xojo permet de générer une application desktop (Windows, Mac, Linux), mobile (Android, iOS), web et Raspberry Pi. Xojo génère des applications natifs pour chaque plateforme cible. Les apps sont indépendantes de Xojo.

Payant
Xojo est payant. L'EDI peut être téléchargé gratuitement et utilisé pour programmer et exécuter des projets. Mais, pour faire un déploiement (build), il faut avoir une licence.

Langage

Caractéristiques
Insensible à la casse: nom variables et mot-clés.
A typage fort: chaque variable doit être typé.
Commenté par // ou '
Les string utilisent "", il faut doubler pour afficher "
L'index d'un tableau commence à 0 (comme C)

Variables
Les variables s'introduisent avec le mot clé Var et doivent avoir un type. 

//Déclaration de variable
Var nomEmploye As String
Var uneDate As DateTime
Var uneVar, uneDeuxieme, uneTroisieme As Double

'Instanciation de variable
nomEmploye = "Rakoto"
uneDate = New DateTime(2024, 2, 23)

'Opérations sur une variable
+ Ajout (nombres) ou concatenation (string)
- Soustraction
* Multiplication
/ Division réelle
\ Division entière
Mod Modulo
= Affectation de valeur à une variable

Type de données
String: chaine de caractères, "" par défaut.
Integer: nombre entier, positif ou négatif, 0 par défaut, peut s'écrire en Int32 ou Int64 aussi pour plus de précision.
Double: nombre réel, O.O par défaut.
Boolean: bouléen (True ou False), False par défaut.

Conversion entre types
String=>Integer: nomVar.ToInteger
String=>Double: nomVar.ToDouble
Integer=>String: nomVar.ToString
Double=>String: nomVar.ToString

ToString peut recevoir des formats d'affichage en paramètre.
"#%" pour un pourcentage (fois 100 puis %)
"###,###.00" pour 2 chiffres après virgule et séparateur décimal
"###,###.##" pour 2 chiffres après virgule si existe
"00000" pour 5 chiffres avec 0 si absent (5 => 00005)

Traitement des chaines de caractères 

Var s As String  'déclaration

unEntier.ToString  'conversion entier=>chaine

If "ceci"="CECI" Then   'renvoie True (car Xojo est case insensitive)

Dates & Heures

Une dates et heure se manipule avec l'objet DateTime. Xojo considère généralement une date comme une simple variable. 

Var unJour As New DateTime(2024, 2, 18)
Var unJourPrecis As New DateTime(2024, 2, 18, 10, 0, 0)  'avec heure
Var ceJour As DateTime = DateTime.Now

Var leJour, leMois, lAnnee, leJourDeSemaine, leJourDeLAnnee, leSemaineDeLAnnnee As Integer
leJour = ceJour.Day
leMois = ceJour.Month  'Janvier=1, Février=2...
lAnnee = ceJour.Year
leJourDeSemaine = ceJour.DayOfWeek  'Dimanche=1, Lundi=2...
leJourDeLAnnee = ceJour.DayOfYear
leSemaineDeLAnnnee = ceJour.WeekOfYear
If uneDate.SecondsFrom1970 > autreDate.SecondsFrom1970 Then ...  'Comparer deux dates
ceDate = ceJour.ToString  'Converti en texte

Boucles

Boucle For ... Next 

Var counter As Integer
For counter = 1 To 10  
    MessageBox(counter.ToString)
Next

'Variantes possibles:
For counter = 10 DownTo 1 ...  'Compter du haut vers le bas
For counter = 1 To 10 Step 2 ...  'Sauter des valeurs
Boucle Do ... Loop ... Until 
Do Until x > 10  'La boucle peut ne jamais s'exécuter
    x = x + 1
Loop

'Variante où la boucle s'exécute au moins une fois
Do
    x = x + 1
Loop Until x > 10
Boucle While ... Wend 
While x < 10
    x = x + 1
Wend
ATTENTION: A chaque itération, les boucles réévaluent les conditions de sortie. Donc, éviter de mettre une fonction car elle est appelée à chaque fois. Au lieu de ça, affecter à une variable et utiliser la variable comme limite.

Débogage

Pour arrêter l'exécution du programme à un endroit du code donné et voir les valeurs dans les écrans de debug: 

Break

On peut aussi utiliser des points d'arrêt.
Pour cela, cliquer sur l'espace à gauche de la ligne de code où on veut s'arrêter. Un marqueur rouge s'affiche dans ce cas. On peut mettre autant de point d'arrêt qu'on veut. On peut ensuite continuer ou continuer pas-à-pas l'exécution du programme. Pour supprimer d'un coup tous les points d'arrêt, aller dans menu Projet > Breakpoint > Clear all.
