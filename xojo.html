    Caractéristiques

    VB-like
    Xojo s'appuie sur une variante propriétaire du langage Visual Basic. C'est d'ailleurs une sorte de suite à VB6, avec les mêmes philosophies en terme de création d'interface et de gestion de projet.

    Format propriétaire
    Xojo se programme avec son IDE. Les formats de fichiers sont propriétaires et le projet n'est pas tout texte comme d'autres langages de programmation ou framework.

    EDI multiplateforme
    L'EDI de Xojo est disponible sur Windows, Mac et Linux. L'EDI est programmé par Xojo lui-même.

    Cible multiplateforme
    Xojo permet de générer une application desktop (Windows, Mac, Linux), mobile (Android, iOS), web et Raspberry Pi. Xojo génère des applications natifs pour chaque plateforme cible. Les apps sont indépendantes de Xojo.

    Payant
    Xojo est payant. L'EDI peut être téléchargé gratuitement et utilisé pour programmer et exécuter des projets. Mais, pour faire un déploiement (build), il faut avoir une licence.

     

    Langage

    Caractéristiques
    Insensible à la casse: nom variables et mot-clés.
    A typage fort: chaque variable doit être typé.
    Commenté par // ou '
    Les string utilisent "", il faut doubler pour afficher "
    L'index d'un tableau commence à 0 (comme C)

    Variables & Propriétés
    Les variables s'introduisent avec le mot clé Var et doivent avoir un type. Des variables spéciaux, appelées Propriétés, existent et sont généralement lié à la fenêtre principale pour être accessible partout dans les événements et méthodes de la fenêtre et des composants. 

    //Déclaration de variable
Var nomEmploye As String
Var uneDate As DateTime
Var uneVar, uneDeuxieme, uneTroisieme As Double

'Instanciation de variable
nomEmploye = "Rakoto"
uneDate = New DateTime(2024, 2, 23)

'Opérations sur une variable
+ Ajout (nombres) ou concatenation (string)
- Soustraction
* Multiplication
/ Division réelle
\ Division entière
Mod Modulo
= Affectation de valeur à une variable

    Type de données
    String: chaine de caractères, "" par défaut.
    Integer: nombre entier, positif ou négatif, 0 par défaut, peut s'écrire en Int32 ou Int64 aussi pour plus de précision.
    Double: nombre réel, O.O par défaut.
    Boolean: bouléen (True ou False), False par défaut.

    Conversion entre types
    String=>Integer: nomVar.ToInteger
    String=>Double: nomVar.ToDouble
    Integer=>String: nomVar.ToString
    Double=>String: nomVar.ToString

    ToString peut recevoir des formats d'affichage en paramètre.
    "#%" pour un pourcentage (fois 100 puis %)
    "###,###.00" pour 2 chiffres après virgule et séparateur décimal
    "###,###.##" pour 2 chiffres après virgule si existe
    "00000" pour 5 chiffres avec 0 si absent (5 => 00005)

    Dates & Heures

    Une dates et heure se manipule avec l'objet DateTime. Xojo considère généralement une date comme une simple variable. 

    Var unJour As New DateTime(2024, 2, 18)
Var unJourPrecis As New DateTime(2024, 2, 18, 10, 0, 0)  'avec heure
Var ceJour As DateTime = DateTime.Now

Var leJour, leMois, lAnnee, leJourDeSemaine, leJourDeLAnnee, leSemaineDeLAnnnee As Integer
leJour = ceJour.Day
leMois = ceJour.Month  'Janvier=1, Février=2...
lAnnee = ceJour.Year
leJourDeSemaine = ceJour.DayOfWeek  'Dimanche=1, Lundi=2...
leJourDeLAnnee = ceJour.DayOfYear
leSemaineDeLAnnnee = ceJour.WeekOfYear
If uneDate.SecondsFrom1970 > autreDate.SecondsFrom1970 Then ...  'Comparer deux dates
ceDate = ceJour.ToString  'Converti en texte

    Boucles

    Boucle For ... Next 

    Var counter As Integer
For counter = 1 To 10  
    MessageBox(counter.ToString)
Next

'Variantes possibles:
For counter = 10 DownTo 1 ...  'Compter du haut vers le bas
For counter = 1 To 10 Step 2 ...  'Sauter des valeurs
    Boucle Do ... Loop ... Until 
    Do Until x > 10  'La boucle peut ne jamais s'exécuter
    x = x + 1
Loop

'Variante où la boucle s'exécute au moins une fois
Do
    x = x + 1
Loop Until x > 10
    Boucle While ... Wend 
    While x < 10
    x = x + 1
Wend
    ATTENTION: A chaque itération, les boucles réévaluent les conditions de sortie. Donc, éviter de mettre une fonction car elle est appelée à chaque fois. Au lieu de ça, affecter à une variable et utiliser la variable comme limite.

    Sortie prématurée d'une boucle: 
    Continue  'Passe directement à l'itération suivante
Exit  'Sort directement de la boucle

     

    Méthodes & Fonctions

    Méthodes

    Pour ajouter une nouvelle méthode, aller dans menu Insert> Method, puis indiquez les valeurs des paramètres attendues (nom, paramètres, type de retour, portée).

    Paramètres: 

    'Déclaration de paramètres (dans UI):
nomFiltre As String  'une seule param
nomFiltre As String, avecParam As Boolean  'plusieurs params
nomFiltre As String = ""  'avec valeur par défaut

'Appel de méthode:
maMethode  'sans param, ou avec param par défaut
maMethode("filtre")  'avec param

    Fonctions

    Une fonction est une méthode avec valeur de retour (le champ type de retour doit être indiqué à la création de la méthode). La valeur de retour est renvoyé par le mot-clé Return. 
    Return valeurRetour  'retourner valeur dans fonction
maVal = maFonction  'appel d'une fonction pour avoir sa valeur

     

    Tableaux

    Un tableau est une liste d'éléments ayant le même type. Dans Xojo, l'index d'un tableau commence à 0. 

    'Déclarer un tableau
Var maListe() As String  'tableau de chaine à taille variable
Var maListe(10) As String  'tableau de chaine à taille fixe

'Utiliser un tableau
valeur = maListe(0)  'récupérer un élément
For Each Row As String In maListe ... Next  'parcourir les éléments de la liste
maListe(2) = "Valeur"  'mettre valeur
maListe.Add("Valeur")  'ajouter un élément, count augmente
maListe.AddAt(1, "Valeur")  'ajouter un élément à la 2ème position, count augmente
maListe.RemoveAt(1)  'supprime le 2ème élément, count diminue
maListe.RemoveAll  'supprime tous les éléments
count = maListe.Count  'nombre d'éléments du tableau
lastIndex = maListe.LastIndex  'indique l'index du dernier élément (fonction), -1 si vide
maListe.Sort  'ordonner les contenus
maListe.Shuffle  'réordonner aléatoirement les contenus
maListe = unTexte.ToArray(",")  'split d'un texte par ","
unTexte = String.FromArray(maListe, ",")  'join d'un texte par ","

     

    Dictionnaire (clé/valeur)

    Un dictionnaire est un tableau associatif clé/valeur. Dans un dictionnaire, la valeur peut être de type différent. 

    'Déclarer un dictionnaire
Var pairs As New Dictionary  'liste clé/valeur

'Utiliser un dictionnaire
pairs.Value("Cle1") = "Valeur 1"  'ajouter une valeur (ou modifier si clé existant)
maValeur = pairs.Value("Cle1")  'récupérer valeur
maValeur = pairs.Lookup("Cle possible", "Valeur si clé non trouvé")  'récupérer valeur si clé trouvé, sinon valeur par défaut
If pairs.HasKey("Cle") Then ...  'check si clé existe
pairs.Remove("Cle") 'supprime l'élément
pairs.RemoveAll 'supprime tous les éléments

    TextField

    Un TextField est une zone de saisie avec une seule ligne. 

    Var s As String
s = Me.Text  'Récupérer valeur
Me.Text = "Ce texte là"  'Assigner une valeur
selection = Me.SelectedText  'Texte séléctionné
selectionPos = Me.SelectionStart  'Position (index) texte séléctionné
selectionLen = Me.SelectionLength  'Nombre caractères texte séléctionné

'Event courants
Opening  'Lorsque le composant est créé

    Un TextArea est une zone de saisie avec plusieurs lignes. Le contenu est représenté par un texte séparé par EndOfLine.

s = composant.Value  'Récupérer valeur

selectionPos = Me.SelectionStart  'Position (index) texte séléctionné

'Event courants
Opening  'Lorsque le composant est créé

    ListBox

    Un ListBox est un composant graphique simulant une table. 

    Me.AddRow("Une ligne")  'Ajouter une ligne de valeur
Me.AddRowAt(1, "Une ligne")  'Insère une ligne de valeur
Me.RemoveAllRows  'Vider la liste

'Ajout rows sur un listbox à plusieurs colonnes
Me.AddRow("Valeur 1")
Me.CellTextAt(Me.LastAddedRowIndex, 1) = "Valeur 2"
Me.CellTextAt(Me.LastAddedRowIndex, 2) = "Valeur 3"

'Modifier valeur cell donné
Me.CellTextAt(1, 2) = "Entrepreneur"  '2ème row, 3ème colonne

'Sur evenement SelectionChanged
If Me.SelectedRowIndex <> -1 Then  'Index de la ligne séléctionnée
    sSelection = Me.SelectedRowValue  'Valeur séléctionnée
End If

'Propriétés courants
ColumnCount  'nombre de colonnes
HasHeader  'affiche une entête ?
InitialValue  'texte du header, séparé par des tabs, sur la première ligne

'Event courants
Opening  'Lorsque le composant est créé
SelectionChanged  'Sélection d'une ligne
DoublePressed  'Double-clic

    Débogage

    Pour arrêter l'exécution du programme à un endroit du code donné et voir les valeurs dans les écrans de debug: 

    Break

    On peut aussi utiliser des points d'arrêt.
    Pour cela, cliquer sur l'espace à gauche de la ligne de code où on veut s'arrêter. Un marqueur rouge s'affiche dans ce cas. On peut mettre autant de point d'arrêt qu'on veut. On peut ensuite continuer ou continuer pas-à-pas l'exécution du programme. Pour supprimer d'un coup tous les points d'arrêt, aller dans menu Projet > Breakpoint > Clear all.
